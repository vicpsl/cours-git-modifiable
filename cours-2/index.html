1. Les branches

  • Les branches sont comme des Sauvegarder sous pour lensemble du dpt

    • Sauf quon peut les joindre/fusionner plus facilement !

  • La branche par dfaut sappelle master (ou main)

  • Les branches offrent la possibilit de travailler sur diffrents problmes en parallle :

    • Une branche principale master
    • Une branche bug 1
    • Une branche bug 2 parce que celui-ci est plus press
    • etc

2. Les branches : vocabulaire et commandes

  • Crer une branche : git branch [nom de la branche]
  • Se dplacer dans une branche : git checkout [nom de la branche]
  • Crer et se dplacer sur une nouvelle branche en mme temps : git checkout -b [nom de la branche]
  • Fusionner une branche : git merge [branche dont on veut les modifications] (on fusionne toujours depuis la branche rceptrice, celle sur laquelle on veut continuer de travailler ensuite)

  5. Rsolution dun conflit (simple) :
    1. Ouvrir le fichier (exemple.xml)
    <TEI xmlns="http://www.tei-c.org/ns/1.0">
              <teiHeader>
                  <fileDesc>
                     <titleStmt>
            <<<<<<< HEAD
                        <title>Epigrammes</title>
                        <author>Martial</title>
            =======
                        <title>Tom</title>
                        <author>Riddle</title>
            >>>>>>> changement_texte
                     </titleStmt>
            ...
    Comme vous pouvez le voir, Git a ajout une syntaxe comprenant sept caractres infrieurs (<<<<<<<) et sept caractres suprieurs (>>>>>>>), spars par sept signes gal, =======.
    Ceux-ci peuvent tre recherchs laide de votre diteur pour trouver rapidement o les modifications doivent tre apportes.

    Il y a deux sections dans ce bloc :
  • Les caractres <<<<<<< indiquent les modifications de la branche actuelle (dans ce cas, HEAD, qui est un autre mot pour votre branche actuelle, dans notre cas master), et les signes ======= indiquent la fin de la premire section.
  • La deuxime section est celle o les modifications proviennent de la tentative de fusion, ici changement_texte: il commence aprs les signes gal et se termine par les signes >>>>>>>.

    1. Editer le fichier en ne conservant que le contenu final souhait, par exemple :
    <TEI xmlns="http://www.tei-c.org/ns/1.0">
              <teiHeader>
                  <fileDesc>
                     <titleStmt>
                        <title>Tom</title>
                        <author>Riddle</title>
                     </titleStmt>
            ...
    1. Enregistrer le changement (git add exemple.xml puis git commit -m rsolution du conflit fichier TEI)
    2. Vrifier le rsultat
